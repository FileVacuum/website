📖 FileVacuum Help Pages

FileVacuum is a macOS application that helps you automatically offload file from your recording devices (phones, cameras, recorders). It runs in the background and waits for a new device to connect to your laptop at the default file path of “/Volumes”

Use the side menu to choose which part of the application you’d like to learn more about.

⏱ Hours Saved

This is the number of hours using FileVacuum has saved you.

In the below picture the “hours saved” is: “0.020”

The "Hours Saved" total is calculated by timing the vacumming's copy/zipping of files from attached devices and then adding 8 seconds.
This simulated 8 seconds accounts for the time it would take to open the attached drive's folder, open the folder inside with the record files, click to copy them, click to the local computer folder where you want to paste them, and then finally right click to actually paste them into that local folder.

🎛 Options*

Options allow you to control FileVacuum's behavior:

  • "eject drive after" option
    This option ejects the attached drive after it's been vacuumed.

  • "delete originals" option
    This option deletes the files on the attached drive after it's been vacuumed.

  • "backup to external" option
    This option moves files off the computer running FileVacuum and onto an external storage drive.

  • "zip" option
    This option zips the vacuumed files.

  • "notifications" option
    This option toggles whether FileVacuum should show notifications.

  • "notifications" option
    This option toggles whether FileVacuum should show notifications.
    An example is notifying when an attached drive's files are finished being vacuumed.

  • "auto launch" option
    This option toggles whether FileVacuum should start automatically when the computer powers on.

  • "ask first" option
    This option toggles whether FileVacuum asks if a newly attached drive should be vacuumed or not.

If a Option is changed while FileVacuum is vacumming,
a "Pending Restart..." message will be displayed and FileVacuum will be restarted after vacuuming finishes.
